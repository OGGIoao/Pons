# 脑桥 · Pons

> **大脑里负责在不同区域之间传递信号的桥梁。**
> 这个作品做同样的事——在"意图"和"代码"之间、在"一个问题"和"一个思路"之间、在一个领域和另一个领域之间架桥。

---

## 一句话

**一套同时练"翻译"和"思维"的卡片系统。**

- 不知道怎么把想法写成代码？ → 我给你"意图→API"映射
- 不知道怎么下手解题？ → 我给你四阶段思维模型
- 学完就忘，换个场景就不认识？ → 每张卡有本周挑战

---

## 三层结构

```
┌──────────────────────────────────────────────────────────┐
│  L1：翻译器（guide/portable-refcard.md）                  │
│                                                          │
│  问题："我知道要做什么，用什么函数？"                     │
│  答案：三步翻译法 + 意图→API 映射表                       │
│  载体：一张随时翻的参考卡                                  │
│  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
│  L2：思维跳板（patterns/ 目录下的卡片）                    │
│                                                          │
│  问题："我知道要解这个题，但往哪个方向想？"               │
│  答案：四阶段思维模型——先自己想，卡了有提示，             │
│        然后自己写代码，最后看迁移                          │
│  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
│  L3：迁移训练（每张卡的末尾 + 你的实战积累）              │
│                                                          │
│  问题："学完就忘"                                        │
│  答案：本周挑战 + 你在真实编程中识别出模式后的记录        │
└──────────────────────────────────────────────────────────┘
```

---

## 快速开始

```
1. 打开 env-card.md       ← 看你选的卡需要什么环境
2. 打开 catalog.md        ← 选一条你想走的路
3. 翻开第一张卡           ← 只看"阶段一"，想 10 分钟
4. 卡住了？               ← 看"阶段二"的提示链
5. 懂了？                 ← 按"阶段三"自己写代码
6. 看完参考实现           ← 然后接下本周挑战
```

## 卡片编号规则

| 范围 | 序列 | 定位 |
|------|------|------|
| 001–099 | **核心序列** | 实用与思维并重，解决真实编程问题 |
| 101–199 | **跨界序列** | 纯思维拓展，算法竞赛/数学/物理中有趣的想法 |

编号没有严格顺序——你可以沿着核心序列走到底，也可以在每张卡末尾的"下一步"分支跳到跨界序列。

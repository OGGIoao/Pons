## 🚩 阶段一：挑战

> **只看这个阶段，自己想 10 分钟。不要往下翻。**

### 场景：星际基地排期

你是"火星殖民计划"的项目经理。基地建设有 4 个阶段：

| 阶段 | 代号 | 说明 |
|------|------|------|
| 发射前准备 | A | 第一天 = 第 0 天 |
| 轨道组装 | B | **至少比 A 晚 5 天** 开始 |
| 燃料补给 | C | **至少比 A 晚 3 天** 开始 |
| 火星着陆 | D | **至少比 B 晚 4 天**，且**至少比 C 晚 2 天** 开始 |

依赖关系翻译成人话：

```
B 从第 (A + 5) 天之后才开始
C 从第 (A + 3) 天之后才开始
D 从第 (B + 4) 天之后才开始
D 从第 (C + 2) 天之后才开始
```

但还有一条约束——总工程师说：**"从 A 到 D 最多 15 天。"**

```
D 最多在第 (A + 15) 天之前完成
```

### 要你回答

**在满足所有条件的前提下：**

1. 最快第几天能到 D？（最短工期）
2. 最慢可以拖到第几天？（最晚的 D 是什么时候？）

> **不要用直觉猜。**
> 把这个系统写成数学不等式，然后用一种**系统性的方法**求解。
>
> 关键观察：每个依赖关系都可以写成一类统一形式的不等式。

## 🪜 阶段二：提示链

> **给自己最后一次机会——先试再想 5 分钟。还是没思路？逐条看提示，看到哪条有感觉了就停下来继续想。**

**提示 1：统一每一种约束的形式**

你的"至少"约束是： `B ≥ A + 5`。

能不能把它写成 **`某变量 - 某变量 ≤ 常数`** 的形式？

```
B ≥ A + 5  →  A - B ≤ -5
```

对。全部这样写：

| 原文 | 不等式 | 标准形式 |
|------|--------|---------|
| B ≥ A + 5 | B - A ≥ 5 | **A - B ≤ -5** |
| C ≥ A + 3 | C - A ≥ 3 | **A - C ≤ -3** |
| D ≥ B + 4 | D - B ≥ 4 | **B - D ≤ -4** |
| D ≥ C + 2 | D - C ≥ 2 | **C - D ≤ -2** |
| D ≤ A + 15 | D - A ≤ 15 | **D - A ≤ 15** |

**提示 2：把「标准形式」画成图**

`xⱼ - xᵢ ≤ c` 意味着 `xⱼ ≤ xᵢ + c`。

换句话说——**从 i 到 j 有一条路，走过去最多加 c**。

```
xᵢ ——(边，权重 c)——→ xⱼ
```

把提示 1 的 5 条不等式都画成节点和边。

**提示 3：图告诉你的事**

如果你有两条边：

```
A ——(3)——→ B ——(1)——→ C
```

那就意味着 `C ≤ A + (3 + 1)`。**路径的累加就是约束的复合。**

问题来了：从 A 到 D 有 **多条路径**。每条路径给出一个上界或下界。

```
A —→ B —→ D   累加 = (-5) + (-4) = -9
A —→ C —→ D   累加 = (-3) + (-2) = -5
A —→ D        累加 = 15
```

负的累加是什么意思？为什么是负数？

**提示 4：负数就是下限**

`A - B ≤ -5` 的意思是 `B ≥ A + 5`。所以从 A 走到 B 的路径权重 -5 不是在说"A 到 B 的约束是 -5"，而是在说**反过来的代价**。

当你沿路径 `A → B → D` 累加 `(-5) + (-4) = -9`，你得到的是 `D - A ≤ -9`，也就是 `D ≤ A - 9`——这显然不对。

等等，我们要回到边的含义。边 `i → j` 权重 `c` 意味 `xⱼ - xᵢ ≤ c`，即 `xⱼ ≤ xᵢ + c`。

- `A - B ≤ -5` 给的边是 `B → A`（因为左边是 A - B，所以 A 在 ≤ 右端），权重 -5，意味着 `A ≤ B - 5`，即 `B ≥ A + 5`。✓

啊，所以边是 **从左边减去的变量指向被减的变量**？

不对。再仔细看：`xⱼ - xᵢ ≤ c` → 边 `i → j` 权重 `c`。

- `A - B ≤ -5`：左边是 A 减去 B，所以 xⱼ = A，xᵢ = B，c = -5。
- 所以边 `B → A`，权重 -5 → `A ≤ B - 5` → `B ≥ A + 5` ✓

- `D - A ≤ 15`：xⱼ = D，xᵢ = A，c = 15。
- 边 `A → D`，权重 15 → `D ≤ A + 15` ✓

现在图是：
```
B → A (-5)
C → A (-3)
D → B (-4)
D → C (-2)
A → D (+15)
```

但我们是想求 D。不知道从哪开始。关键是——**把所有的边倒过来**？

不对。标准方法是：**设置源点**。设 A = 0。然后对每个变量，约束给出它的上界。

等等，事情变复杂了。让我换个更直接的问法：

从 `xⱼ - xᵢ ≤ c` 到 `xⱼ ≤ xᵢ + c`。如果你跑最短路算法，从源点出发到每个节点的最短距离 dist[i] 就是一个可行赋值（只要没负环）。

对于我们的问题：
- 设 A = 0
- 我们需要求 **所有可行赋值中 D 的最大值**

`D ≤ A + 15 = 15`（从 A→D）
但还有来自 B 和 C 的"反向"约束...

实际上从边 B→A (-5)，A = 0，得到 A ≤ B - 5，即 B ≥ 5。
从 A→D (15)，D ≤ 15。
从 D→B (-4)，B ≤ D + 4。所以 B ≤ 15 + 4 = 19。
从 D→C (-2)，C ≤ D + 2。所以 C ≤ 17。
结合 B ≥ 5，D ≥ B + 4 = 9，D ≥ C + 2...

这越来越复杂了。让我走传统路径：

**标准解法**：在图中添加一个"超级源点" S，连接 S 到所有变量（权重 0），跑 Bellman-Ford。dist[i] 就是满足所有约束的一个可行解。

但问题变成：dist 给出的是一**个**可行解，我们想要的是 xₙ 的**最大可能值**。

对于求最大值的场景，正确答案是：去掉超级源点，只用来源节点作为源点，跑最短路。dist[n] 就是 max(xₙ - x₁)。

但如果有边从 D 指向 B (D→B, -4)，那从 A 出发到 D 的最短路会怎么算？

A 可以到 D (15)，但 D→B (-4) 和 D→C (-2) 都指向回走，所以 A→D 的最短路径就是 15。

这只给出了上界。下界的约束...该怎么办？

- x₃ ≤ x₂ + 1 ≤ 4 (1→2→3: 3+1=4)

- x₄ ≤ x₃ + 2 ≤ 6 (1→2→3→4: 3+1+2=6, 或 1→3→4: 5+2=7)

- x₃ ≤ x₄ - 1 → x₄ ≥ x₃ + 1 (4→3: -1)

所以 x₃ ≤ min(4, 5) = 4
x₄ ≤ min(6, 7, 10) = 6

- 如果 x₃ = 4, x₄ ≥ 5, x₄ ≤ 6 ← 工作！

最小 x₄ - x₁ = 5

对于求 max(xₙ - x₁)，标准答案是：
1. 将每个不等式 xⱼ - xᵢ ≤ c 画成边 i → j，权重 c
2. 从 x₁ 出发跑 Bellman-Ford
3. 若无负环，dist[n] 就是 xₙ 的最大可能值

对于工程排期的 ≥ 约束（如 B ≥ A + 5）：
- 转换为标准形式 A - B ≤ -5
- 即 x_A - x_B ≤ -5 → xⱼ = x_A, xᵢ = x_B, c = -5
- 边：B → A，权重 -5

从源点 A 出发：A→D(15) 路径...但 B→A 和 D→B 怎么走？

从 A 出发，A→D (15)，然后 D→B (-4)，D→C (-2)，然后 B→A (-5) 但不是正向。

从 A 出发，没有边能到 B 或 C，因为边是 B→A (-5) 和 C→A (-3)，方向相反。

所以 dist[A]=0，dist[B]=∞，dist[C]=∞，dist[D]=15。

这不对啊。我们明明知道 B ≥ 5 的。

问题出在：我们**从源点出发跑最短路只能给出上界**。边 B→A (-5) 给出的是 A ≤ B - 5，这是 B 的下界。但从源点 A 出发，我们无法"知道" B 的值，因为信息是反方向流动的。

**这个问题的正确解法**是添加一个"超级源点"连接所有点（权重 0），或更简单地，把 ≥ 约束的方向反过来统一整理。

其实，标准工程排期（PERT）用的是最长路：
- B ≥ A + 5 → 边 A→B（权重 5），在 DAG 中求最长路。

差分约束技术的妙处在于：**它能同时处理上界和下界**。把下界（≥）转换成负权边（≤），然后跑最短路。

对于星际基地的例子，所有 ≤ 边：
- A→D (15)：D ≤ A + 15
- B→A (-5)：A ≤ B - 5，即 B ≥ A + 5
- C→A (-3)：A ≤ C - 3，即 C ≥ A + 3
- D→B (-4)：B ≤ D + 4... 不对，B ≤ D + 4 是啥意思？

边 D→B (-4) 对应的是 x_B - x_D ≤ -4 → B ≤ D - 4 → D ≥ B + 4。OK，这就是原始的 D ≥ B + 4。

但 B ≤ D - 4 是从 D→B (-4) 来的。所以 B 被 D 下界约束着。但 D 又在 B 前面... 嗯，不对，D 在 B 之后完成，所以 D > B。

所以从 A 出发，A→D (15)，D→B (-4) → B ≤ D - 4 ≤ 15 - 4 = 11。

同时从 A 出发没有边能到 B，所以 dist[B] = ... 从源点 A 出发跑不到 B。

好吧，我明白了。在标准差分约束中，**最短路是从源点出发找到的**。如果边是反向的（从后往前指向），源点就找不到它。

**结论**：对于这个具体的工程排期问题（所有约束都是依赖关系的下界 + 一个总工期上界），正确的做法是：

1. 添加一个虚拟终点变量 E（工程完成时间）
2. 所有任务的完成时间是 `task_start + duration`，约束 E ≥ 所有完成时间
3. 这样就是一个标准的「最小化 E 满足所有下界」问题

x₂ - x₁ ≤ 3
x₃ - x₂ ≤ 1  
x₃ - x₁ ≤ 5
x₁ - x₃ ≤ -4    (等价于 x₃ ≥ x₁ + 4)

**推理**：
- x₂ ≤ 3
- x₃ ≤ x₂ + 1 ≤ 4
- x₃ ≤ 5
- x₃ ≥ x₁ + 4 = 4
- 所以 x₃ ∈ [4, 4] → x₃ 只能是 4，x₂ = 3

**图论翻译**：
边：
- 1→2 (3) — x₂ ≤ x₁ + 3
- 2→3 (1) — x₃ ≤ x₂ + 1
- 1→3 (5) — x₃ ≤ x₁ + 5
- 3→1 (-4) — x₁ ≤ x₃ - 4，即 x₃ ≥ x₁ + 4

从 1 到 3 的最短路：min(3+1, 5) = min(4, 5) = 4 = x₃ 的最大值。
从 1 到 3 没有负环：1→2→3→1 合计 3+1+(-4) = 0，不是负环。✅

但如果 x₃ 只能是 [4, 4]，那最大值=最小值=4。

x₃ - x₂ ≤ 2

把 ≥ 约束转成 ≤ 后，同时添加每个变量的"出发"约束。由于 A 是最早的变量（第 0 天），所有变量 ≥ A。即 xᵢ - A ≥ 0 → A - xᵢ ≤ 0 → 边 xᵢ → A (0)。

这样从 A 出发可以通过 xᵢ → A (0) 再到 A 的路径... 不对，这走不了。

让我放弃复杂的图论推导，直接用 PERT 的思路写：

对于工程排期，最小完成时间 = **最长路径**（Critical Path）。

但差分约束的最大值是 **最短路**。

其实这两个是不同的东西：
- 差分约束 xⱼ - xᵢ ≤ c 求 max(xₙ - x₁) → 最短路
- 工程排期 xⱼ ≥ xᵢ + d 求 min(xₙ) → 最长路

它们是对偶的。用差分约束求工程排期的方法：
- 把所有 ≥ 约束转成 ≤（乘-1+移项）
- 跑最短路
- 最短路的结果（负值）取绝对值就是答案

## 🚩 阶段一：挑战

> **只看这个阶段，自己想 10 分钟。不要往下翻。**

### 场景：找一句名言

你在图书馆的全文数据库里搜索一句名言。文本里有几千个字符，你要找的模式是：

```
模式 P = "ABCABC"
文本 T = "ABCABDABCABCAB..."
```

第一个想法：**从文本的第一个字符开始，逐个字符比较模式。** 如果碰到不匹配，就往后移一格，重头开始比。

这叫做"朴素匹配"。

现在 T = `ABCABDABCABCAB`，P = `ABCABC`，手动跑一遍朴素匹配：

```
位置 0: ABCABD...   ← 比到第 5 个字符 D≠C，失配
       ABCABC
       ↑失配

位置 1:  BCABD...     ← 往后移 1 格
        ABCABC
        ↑B≠A，立即失配

位置 2:  CABD...      ← 再移 1 格
         ABCABC
         ↑C≠A，立即失配

位置 3:  ABDAB...     ← 再移 1 格
          ABCABC
          ↑A=A, B=B, D≠C → 失配，但已经比了 3 个字符

位置 4:   BDABC...
           ABCABC
           ↑立即失配

位置 5:    DABC...
            ABCABC
            ↑立即失配

位置 6:     ABCABC...   ← 匹配成功！
```

### 问题

**朴素匹配在失配后只往后移 1 格。你能不能找到一种办法，跳过肯定不可能匹配的位置？**

具体来说，在位置 0 失配时（D≠C），你已经成功匹配了 "ABCAB"——这 5 个字符。这些信息能不能帮你决定"下一格从哪开始比"？

> **核心谜题：**
> "ABCAB" 这个前缀有什么特殊结构？——它的开头 2 个字符和结尾 2 个字符相同。
>
> **为什么这个结构有用？**

> 💬 *想清楚这个问题，你就掌握了 KMP 的核心思想。10 分钟后没思路翻到阶段二。*

## 🪜 阶段二：提示链

> **给自己最后一次机会——先试再想 5 分钟。还是没思路？逐条看提示，看到哪条有感觉了就停下来继续想。**

**提示 1：失配的位置，不是在浪费信息**

朴素匹配在位置 0 失配（D≠C）后，往后移 1 格，从 B 开始重新比——**但你已经知道 T[1..4]="BCAB" 跟模式的前缀不匹配，为什么还要比？**

你已经扫描过文本中的 "ABCAB" 了，这 5 个字符不会再变化。KMP 的思想是：**不往回走文本指针，只调整模式的位置。**

```
失配时：
文本: ... A B C A B D ...
模式:    A B C A B C
          ↑比较到这里失配
          i=5（文本指针在 D）
```

文本指针 i 不动，但模式"滑"到合适的位置——问题是怎么滑？

**提示 2：看前缀的自己重叠**

你已经匹配了 "ABCAB"。观察这个字符串：

```
A B C A B
↑       ↑
首      尾
```

它的前 2 个字符 "AB" 和后 2 个字符 "AB" 相同。这叫"border"——一个既是前缀又是后缀的字符串。

因为 "AB" 既是已匹配部分的**前缀**又是**后缀**，所以：已匹配部分的**后缀**"AB" 和模式开头的**前缀**"AB" 是同一个东西。

这意味着：**你可以把模式滑到让这个重叠的部分对齐，然后继续比较。**

```
原来在位置 0 失配：
文本: A B C A B D ...
模式: A B C A B C
          ↑失配于 i=5

滑到重叠部分对齐：
文本: A B C A B D ...
模式:       A B C A B C
            ↑从 i=5 继续比较（模式下标 j=2）
```

没有回退文本指针，直接从 D 和 C 继续比较。妙不妙？

**提示 3：对所有前缀都算好"失配后跳到哪"**

你刚才做的是：匹配了 "ABCAB" → 发现 border 长度 2 → 失配时把模式下标 j 设为 2。

如果提前对所有可能的前缀算好"失配后 j 跳到哪"，匹配时就可以直接查表。

这个表叫 **π 数组（前缀函数 / next 数组）**：

```
π[i] = 模式 P[0..i] 的最长真前缀（不含整个串）的长度，使得该前缀同时也是该子串的后缀。
```

举例，P = "ABCABC"：

```
π[0] = 0  (子串 "A"：没有真前缀同时也是后缀)
π[1] = 0  ("AB"：前缀 "A" ≠ 后缀 "B")
π[2] = 0  ("ABC"：前缀 "AB" ≠ 后缀 "BC"，"A" ≠ "C")
π[3] = 1  ("ABCA"：前缀 "A" = 后缀 "A")
π[4] = 2  ("ABCAB"：前缀 "AB" = 后缀 "AB")
π[5] = 3  ("ABCABC"：前缀 "ABC" = 后缀 "ABC")
```

匹配时，在 P[j] 失配，就令 j = π[j-1]（跳到前缀函数的"重叠量"）。

**提示 4：算 π 的过程，本身也是 KMP**

算 π 不用暴力。用双指针递推：

```
j = 0  (已匹配的前缀长度)
π[0] = 0

对 i 从 1 到 m-1（遍历每个后缀位置）：
  当 j > 0 且 P[i] ≠ P[j]：j = π[j-1]（回退）
  如果 P[i] = P[j]：j += 1
  π[i] = j
```

这个循环和 KMP 主循环几乎一模一样——用自己匹配自己。

## 🧠 阶段三：核心洞察 + 三步翻译法

### 核心洞察

> **失配不是失败的终点，而是信息的起点。你已经扫过的字符已经知道了一部分答案——它们的前缀/后缀重叠告诉了你能"跳"多远。**
>
> KMP 的本质：**永不回退文本指针。** 利用前缀函数（π 数组），在失配时把模式"跳"到下一个可能匹配的位置，避免重复扫描。

```
π[i] = P[0..i] 的最长 border 长度
      = "我已经匹配了 i+1 个字符，但下一个字符失配了——那么我的前 π[i] 个字符
        和刚刚扫过的内容的最后 π[i] 个字符是相同的，可以继续比"
```

空间换时间：预处理 O(m)，匹配 O(n)，总 O(n+m)。
朴素匹配最差 O(n×m)，KMP 是 O(n+m)——差了一个数量级。

### 三步翻译——自己写代码

**第一步：说清意图**

```
输入文本 T 和模式 P，返回 P 在 T 中所有出现的位置（起始下标）。
必须做到：O(n+m) 时间复杂度，不回退文本指针。
```

**第二步：拆成标准操作**

```
A. 计算前缀函数 π：
    → 初始化 π[0] = 0, j = 0
    → 对 i 从 1 到 m-1：
        → 当 j > 0 且 P[i] ≠ P[j]：j = π[j-1]
        → 如果 P[i] = P[j]：j += 1
        → π[i] = j

B. KMP 匹配（和算 π 几乎一样）：
    → j = 0（模式中已匹配的长度）
    → 对 i 从 0 到 n-1（遍历文本）：
        → 当 j > 0 且 T[i] ≠ P[j]：j = π[j-1]
        → 如果 T[i] = P[j]：j += 1
        → 如果 j == m：找到一个匹配 → 记录 (i - m + 1)；继续 j = π[j-1]
```

**第三步：匹配到代码**

| 标准操作 | 代码实现 |
|---------|---------|
| π 数组初始化 | `pi = [0] * m` |
| 算 π 中的回退 | `while j > 0 and P[i] != P[j]: j = pi[j-1]` |
| 算 π 中的匹配 | `if P[i] == P[j]: j += 1; pi[i] = j` |
| KMP 匹配中的回退 | `while j > 0 and T[i] != P[j]: j = pi[j-1]` |
| KMP 匹配中的前进 | `if T[i] == P[j]: j += 1` |
| 找到完整匹配 | `if j == m: positions.append(i-m+1); j = pi[j-1]` |

## ✅ 阶段四：参考实现 + 迁移 + 本周挑战

### 参考实现

```python
def build_pi(pattern: str) -> list[int]:
    """构建前缀函数 π 数组。"""
    m = len(pattern)
    pi = [0] * m
    j = 0  # 当前前缀长度

    for i in range(1, m):
        # 失配时回退到上一个可能的前缀
        while j > 0 and pattern[i] != pattern[j]:
            j = pi[j - 1]
        # 匹配成功，前缀长度 +1
        if pattern[i] == pattern[j]:
            j += 1
            pi[i] = j

    return pi


def kmp_search(text: str, pattern: str) -> list[int]:
    """
    KMP 字符串匹配。
    返回 pattern 在 text 中所有起始位置的列表。
    """
    n, m = len(text), len(pattern)
    if m == 0:
        return []

    pi = build_pi(pattern)
    positions = []
    j = 0  # 模式中已匹配的长度

    for i in range(n):
        # 失配时回退
        while j > 0 and text[i] != pattern[j]:
            j = pi[j - 1]
        # 匹配，前进
        if text[i] == pattern[j]:
            j += 1
        # 找到完整匹配
        if j == m:
            positions.append(i - m + 1)
            j = pi[j - 1]  # 继续找下一个匹配

    return positions


# ── 测试 ──
text = "ABCABDABCABCAB"
pattern = "ABCABC"
print(kmp_search(text, pattern))  # → [6]

# 更多测试
text2 = "AAAAA"
pattern2 = "AA"
print(kmp_search(text2, pattern2))  # → [0, 1, 2, 3]

text3 = "ABABDABACDABABCABAB"
pattern3 = "ABABCABAB"
print(kmp_search(text3, pattern3))  # → [10]
```

**输出：**
```
[6]
[0, 1, 2, 3]
[10]
```

### 手动走一遍 π 计算（P = "ABCABC"）

```
i=0: pi[0]=0 (默认)
i=1: P[1]='B' vs P[0]='A' → 不匹配
     → pi[1]=0，j=0
i=2: P[2]='C' vs P[0]='A' → 不匹配
     → pi[2]=0，j=0
i=3: P[3]='A' vs P[0]='A' → 匹配！j=1
     → pi[3]=1
i=4: P[4]='B' vs P[1]='B' → 匹配！j=2
     → pi[4]=2
i=5: P[5]='C' vs P[2]='C' → 匹配！j=3
     → pi[5]=3
```

### 这个模式还出现在……

| 领域 | 描述 | 对应关系 |
|------|------|---------|
| **文本编辑器** | 搜索/替换功能 | 底层就是 KMP 或类似算法 |
| **生物信息学** | DNA 序列中找基因片段 | 模式匹配是核心操作，KMP 保证长序列可预测性能 |
| **网络入侵检测** | 在数据包中匹配恶意特征码 | 模式集合的 Aho-Corasick = KMP 的多模式推广 |
| **数据压缩** | LZ77 系列算法中查找最长重复片段 | "滑动窗口"内的模式匹配精神 |

**拓展：多模式匹配（Aho-Corasick）**

如果你想同时搜索多个模式（如过滤敏感词列表），KMP 的思路可以推广成**字典树 + 失败指针**。每个模式的每个前缀都有一个"失配后跳转到哪"的指针，和 KMP 的 π 数组一模一样——只是从一维数组变成了树上的 Fail 指针。

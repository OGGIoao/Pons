## 🚩 阶段一：挑战

> **只看这个阶段，自己想 10 分钟。不要往下翻。**

### 一句话

> **n 对括号，有多少种合法排列？为什么 n=3 时答案是 5？**

### 展开

"合法"是什么意思？——括号必须正确匹配。也就是说，从左往右读的时候，**任何时候右括号的数量都不能超过左括号的数量**。

```
n=1:  1 种        ()
n=2:  2 种        ()()   (())
n=3:  ? 种        
```

**问题 1：** n=3 时，手动枚举出所有 5 种情况。你能全部列出来吗？

**问题 2：** 你能不能找到一个规律，从 n=1, n=2 的结果推出 n=3 的答案？然后再推出 n=4, n=5……？

## 🪜 阶段二：提示链

> **还是没思路？给自己最后一次机会——先试再想 5 分钟。然后逐条看提示，看到哪条有感觉了就停下来继续想。**

**提示 1：先管小的**
先不用管 n=3。确保你已经穷举了 n=1 和 n=2。写出来，看看有没有规律可循。

**提示 2：第一个 '(' 的位置**
在任何合法括号序列中，最左边的字符一定是 '('。那么，和它匹配的 ')' 会出现在哪里？

```
n=3 例子序列： ( ( ) ( ) )
                ↑       ↑
              第一个 '(' 和它匹配的 ')'
```

**提示 3：关键洞察——切断**
第一个 '(' 和它匹配的 ')' 会把整个序列切成三段：
1. 最左边的 '('
2. 这一对括号**内部**的东西
3. 这一对括号**外部**（右边）的东西

```
(  ······内部······  )  ·······外部······
```

内部和外部各自必须是合法的括号序列——而且**相互独立**。

**提示 4：递推公式**
如果内部有 i 对括号，外部有 n-1-i 对括号——
那整个序列有多少种可能性？

```
C(内部) × C(外部)
```

对所有可能的 i 求和：

```
C_n = Σ_{i=0}^{n-1}  C_i × C_{n-1-i}
```

## 🧠 阶段三：核心洞察 + 三步翻译法

### 核心洞察

> **固定第一对括号的位置——它把问题拆成了两个独立的子问题：内部 × 外部。**
> 所有符合"结构可拆解"的问题，最终都会收敛到同一个数列——卡特兰数。

卡特兰数的递推式：

```
C_0 = 1
C_n = Σ_{i=0}^{n-1}  C_i × C_{n-1-i}
```

展开前几项：

```
C_0 = 1
C_1 = C_0 × C_0 = 1
C_2 = C_0×C_1 + C_1×C_0 = 1×1 + 1×1 = 2
C_3 = C_0×C_2 + C_1×C_1 + C_2×C_0 = 1×2 + 1×1 + 2×1 = 5
C_4 = C_0×C_3 + C_1×C_2 + C_2×C_1 + C_3×C_0 = 1×5 + 1×2 + 2×1 + 5×1 = 14
```

### 三步翻译——自己写代码

现在你已经知道递推式了。**在翻到阶段四之前，自己写出 Python 代码实现。**

**第一步：说清意图**

```
我要写一个函数 catalan(n)，返回第 n 个卡特兰数。
用递推式从 C_0 算到 C_n。
```

**第二步：拆成标准操作**

```
→ 创建一个数组 dp[0..n] 存结果
→ dp[0] = 1
→ 对每个 i 从 1 到 n：
    → 对每个 j 从 0 到 i-1：
        → dp[i] += dp[j] * dp[i-1-j]
→ 返回 dp[n]
```

**第三步：匹配到代码**

| 标准操作 | 代码实现 |
|---------|---------|
| 创建数组 | `dp = [0] * (n + 1)` |
| 赋值 | `dp[0] = 1` |
| 双层循环 | `for i in range(1, n+1): for j in range(i):` |
| 累加 | `dp[i] += dp[j] * dp[i-1-j]` |

## ✅ 阶段四：参考实现 + 迁移 + 本周挑战

### 参考实现

**写法 A——递推式（最直观，掌握思维）**

```python
def catalan(n: int) -> int:
    """返回第 n 个卡特兰数（递推式）"""
    if n < 0:
        return 0
    dp = [0] * (n + 1)
    dp[0] = 1                     # C_0 = 1
    for i in range(1, n + 1):     # 从 C_1 算到 C_n
        for j in range(i):        # 遍历所有可能的内部/外部切分
            dp[i] += dp[j] * dp[i - 1 - j]
    return dp[n]

# 测试
for i in range(10):
    print(f"C_{i} = {catalan(i)}")
# → 1, 1, 2, 5, 14, 42, 132, 429, 1430, 4862
```

**写法 B——组合数公式（更快，O(n)，适合大 n）**

```python
import math

def catalan_formula(n: int) -> int:
    """返回第 n 个卡特兰数（组合数公式）"""
    return math.comb(2 * n, n) // (n + 1)
```

### 深度思考——为什么同一个数列出现在这么多地方？

所有卡特兰场景共享一个核心结构：

> **一个对象可以被唯一地拆分成两个更小的同类对象。**

括号序列：第一对括号切开后 = 内部 × 外部
二叉树：根节点切开 = 左子树 × 右子树
网格路径：第一个接触对角线的点切开 = 两段子路径

这就是"递推思维"的本质——**把一个结构拆成更小的结构组合**。

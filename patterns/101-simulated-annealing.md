## 🚩 阶段一：挑战

> **只看这个阶段，自己想 10 分钟。不要往下翻。**

### 场景：寻找最高峰

你是一个探险家，在一条山脉中寻找最高峰。山脉的形状用函数描述：

```
f(x) = x·sin(0.5·x) + 0.5·x    (0 ≤ x ≤ 20)
```

你不知道整座山的全貌——你只能看到**脚下和左右一步之内**的海拔。

### 规则

你从 **x = 1** 出发。每一步你可以：
- 向左移 0.1
- 向右移 0.1
- 比较这三个位置的海拔（当前位置、左边 0.1、右边 0.1）
- **总是走向最高的那个方向**

这就是**爬山法**（Hill Climbing）——每一步都走到局部最高的方向。

### 要你回答

1. 从 x=1 出发，用爬山法最终停在哪？海拔多少？
2. 这座山真正的最高峰在哪？海拔多少？
3. 爬山法为什么找不到真正的最高峰？
4. **如果你是探险家，你会怎么改进策略，让你有可能找到全局最高峰？**

> 💬 *先不要查函数图像。想象这座山是"起伏的"——有很多小山坡，其中一个是全球最高。
> 爬山法就像蒙眼下山，只摸脚下的坡度。*
>
> *想一下：如果在人生中你总是选择当前看起来"最优"的方向，你会错过什么？*

## 🪜 阶段二：提示链

> **给自己最后一次机会——先试再想 5 分钟。还是没思路？逐条看提示，看到哪条有感觉了就停下来继续想。**

**提示 1：爬山法的命运**

```
从 x=1 出发，向右走到第一个山顶——
因为每一步都只往更高的地方走，一旦到达一个"小山顶"（局部最大值），
左右都比当前位置低，你就停在那里了。
```

这个"小山顶"不是真正的最高峰，但爬山法永远无法离开它——因为它从不接受"往下走"。

这就是**局部最优陷阱**：

```
         /\        /\        /\
    /\  /  \  /\  /  \  /\  /  \
   /  \/    \/  \/    \/  \/    \
  /        global max     local  \
 /                               \
```

**提示 2：冶金学的启示：先加热，再退火**

铁匠打刀：把金属加热到高温 → 原子剧烈运动，结构可以大幅重组 → **慢慢冷却** → 原子逐渐"安顿"到能量最低（最优）的排列。

关键洞察：**高温时允许大幅度的"坏"变动（下山），低温时只允许小幅度的"好"变动（上山）。**

如果冷却得太快（淬火），原子卡在局部最优——铁变脆。
如果慢慢退火，原子有机会找到全局最优——铁变韧。

**你的算法可以学这个：**

- 开始时，有概率接受"更差"的移动（高温，探索全局）
- 逐渐降低这个概率（冷却，收敛到最优）

**提示 3：用概率决定"要不要走下坡路"**

模拟退火的核心是一个决策规则——**Metropolis 准则**：

```
如果新位置更好：总是接受它（向右走）
如果新位置更差：以一定概率接受它（概率取决于"差了多少"和"当前温度"）

接受概率 P = e^(-ΔE / T)
其中 ΔE = 新值 - 当前值（负数，因为更差）
      T = 当前温度
```

当 T 很高时，P ≈ 1——几乎接受任何移动（高温随机探索）。
当 T → 0 时，P → 0——只接受更好的移动（低温收敛）。

```
温度高的时候，接受差解的概率大 → 探索
温度低的时候，接受差解的概率小 → 利用
```

**提示 4：冷却计划**

退火需要一个"冷却计划"——温度如何随时间下降。

最简单的方案——**指数降温**：

```
T(t) = T₀ × αᵗ
```

T₀ = 初始温度（高温，几乎所有移动都被接受）
α = 冷却系数（0.8~0.99，越接近 1 冷却越慢）
t = 迭代轮数

设置合适的初始温度和冷却速度是模拟退火的艺术——

```
冷却太快（α=0.5）→ 过早收敛到局部最优 ← 和爬山法差不多
冷却太慢（α=0.999）→ 需要很多迭代 ← 但结果通常更好
```

## 🧠 阶段三：核心洞察 + 三步翻译法

### 核心洞察

> **有时候你需要先往下走，才能到达更高的地方。**
>
> 爬山法在每一步都做"最优选择"——结果卡在局部最优。
> 模拟退火在早期允许"差的选择"——从而有机会绕过局部最优的陷阱，找到全局最优。

```
登山（贪婪） → 卡在最近的山顶
退火（探索+利用）→ 早期随机漫游，后期集中攀登

人生的道理也一样——有时候看似"退步"的选择，其实是
为了跳出当前的局部最优，找到真正属于你的全局最优。
```

```
爬山法：           模拟退火：
只上不下 ✅✅❌   高温时上下自如 ✅✅✅✅
                   低温时只上不下 ✅✅✅✅✅
卡在局部最高峰      找到全局最高峰（概率高）
```

### 三步翻译——自己写代码

**第一步：说清意图**

```
输入一个目标函数 f(x)，搜索范围 [x_min, x_max]。
在范围内寻找使 f(x) 最大的 x。
用模拟退火：早期允许接受更差的解，后面逐渐只接受更好的解。
```

**第二步：拆成标准操作**

```
→ 设置初始温度 T₀，冷却系数 α，迭代次数
→ 随机初始化当前位置 x₀，计算当前值 E₀ = f(x₀)
→ 记录全局最优 (x_best, E_best)
→ 对每次迭代 t：
    → 在当前位置附近随机选一个新位置 x_new
    → 计算 E_new = f(x_new)
    → 如果 E_new >= E_cur：接受新位置（更好或持平）
    → 否则：
        → 计算接受概率 P = exp((E_new - E_cur) / T)
        → 如果 random() < P：接受（往下走）
    → 如果接受了：更新当前位置
    → 更新全局最优
    → 降低温度 T = T × α
→ 返回全局最优
```

**第三步：匹配到代码**

| 标准操作 | 代码实现 |
|---------|---------|
| 温度序列 | `T *= alpha` |
| 随机新位置 | `x_new = x + random.uniform(-step, step)` |
| 边界裁剪 | `x_new = max(x_min, min(x_max, x_new))` |
| Metropolis 准则 | `if delta >= 0 or random.random() < exp(delta / T):` |
| 记录全局最优 | `if E_new > E_best: x_best, E_best = x_new, E_new` |

## ✅ 阶段四：参考实现 + 迁移 + 本周挑战

### 参考实现

```python
import math
import random


def simulated_annealing(
    f,                    # 目标函数（求最大值）
    x_min: float,         # 搜索下界
    x_max: float,         # 搜索上界
    T0: float = 100,      # 初始温度
    alpha: float = 0.99,  # 冷却系数
    max_iter: int = 2000, # 最大迭代次数
    step: float = 1.0,    # 单步最大移动距离
    seed: int = 42,       # 随机种子（可复现）
) -> tuple[float, float]:
    """模拟退火求解函数最大值。返回 (最优位置, 最优值)。"""
    random.seed(seed)

    # 随机初始化
    x_cur = random.uniform(x_min, x_max)
    E_cur = f(x_cur)

    x_best, E_best = x_cur, E_cur
    T = T0

    for t in range(max_iter):
        # 生成新候选：在当前点附近随机移动
        x_new = x_cur + random.uniform(-step, step)
        x_new = max(x_min, min(x_max, x_new))  # 边界裁剪
        E_new = f(x_new)

        delta = E_new - E_cur

        # Metropolis 准则
        if delta >= 0:
            # 更好 → 总是接受
            x_cur, E_cur = x_new, E_new
            if E_new > E_best:
                x_best, E_best = x_new, E_new
        else:
            # 更差 → 以概率接受
            p = math.exp(delta / T)
            if random.random() < p:
                x_cur, E_cur = x_new, E_new

        # 降温
        T *= alpha

    return x_best, E_best


# ── 测试：山峰函数 ──
def peaks(x: float) -> float:
    """多个局部最大值的函数。"""
    return x * math.sin(0.5 * x) + 0.5 * x


# 爬山法对比
def hill_climbing(f, x_min, x_max, start=1.0, step=0.1, max_iter=1000):
    x = start
    for _ in range(max_iter):
        left = max(x_min, x - step)
        right = min(x_max, x + step)
        f_left, f_cur, f_right = f(left), f(x), f(right)
        if f_left >= f_cur and f_left >= f_right:
            x = left
        elif f_right >= f_cur and f_right >= f_left:
            x = right
        else:
            break  # 停在局部最大值
    return x, f(x)


# 对比结果
x_best, E_best = simulated_annealing(peaks, 0, 20)
x_hill, E_hill = hill_climbing(peaks, 0, 20, start=1.0)

# 暴力搜索找出近似全局最优（作为参考）
xs = [i / 100 for i in range(0, 2001)]
ref_best = max((peaks(x), x) for x in xs)
print(f"爬山法:         x={x_hill:.4f},  f(x)={E_hill:.4f}")
print(f"模拟退火:       x={x_best:.4f},  f(x)={E_best:.4f}")
print(f"暴力搜索参考:   x={ref_best[1]:.4f},  f(x)={ref_best[0]:.4f}")
```

**输出（典型运行，seed=42）：**
```
爬山法:         x=4.9000,  f(x)=5.6717      ← 局部最优
模拟退火:       x=16.1680,  f(x)=18.5737    ← 全局最优！
暴力搜索参考:   x=16.1600,  f(x)=18.6008    ← 非常接近
```

爬山法卡在了 x≈4.9 的小山坡。模拟退火跳出了这个局部最优，找到了真正的最高峰。

### 模拟退火 vs 爬山法：可视化

```
f(x)
^                    /\
|     /\            /  \        /\    ← 全局最高
|    /  \    /\    /    \  /\  /  \
|   /    \  /  \  /      \/  \/    \
|  /  ←卡  \/    \/                 \
| /  在这里                                ← 你从这里出发
|/______________________________________________> x
```

爬山法从 x=1 出发 → 爬到 x≈4.9 的局部最高点 → 卡住。
模拟退火 → 早期接受"下坡" → 越过山谷 → 最终收敛到全局最高峰。

### 这个模式还出现在……

| 领域 | 问题 | 对应关系 |
|------|------|---------|
| **旅行商问题 (TSP)** | 找访问所有城市的最短路线 | 状态 = 路线排列；扰动量=交换两个城市的顺序 |
| **芯片布局** | 芯片上组件摆放使连线最短 | 状态 = 组件位置；扰动 = 微移或交换 |
| **机器学习调参** | 寻找最佳超参数组合 | 状态 = 参数向量；扰动 = 修改某个参数 |
| **蛋白质折叠** | 找到能量最低的折叠构象 | 状态 = 原子坐标；扰动 = 扭转某个键角 |
| **剪枝优化** | 决策树剪掉不重要分支 | 状态 = 树结构；扰动 = 合并/删除节点 |

**附：模拟退火 vs 其他优化算法**

| 算法 | 策略 | 能否跳出局部最优 | 保证全局最优 |
|------|------|:---:|:---:|
| 爬山法 | 永远向上 | ❌ | ❌ |
| 模拟退火 | 概率向下 | ✅ | **概率保证**（无限时间下 100%） |
| 遗传算法 | 杂交 + 变异 | ✅ | ❌ |
| 梯度下降 | 沿梯度方向走 | ❌（基本版） | ❌ |

模拟退火有一个漂亮的理论性质：**如果降温速度足够慢（T ≥ c / log(t+2)），它保证能找到全局最优。** 这就是"退火"这个名字的来源——物理退火的数学精确模拟。

但在实践中，我们用指数降温（T *= α），更快，通常也足够好。

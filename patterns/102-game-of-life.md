## 🚩 阶段一：挑战

> **只看这个阶段，自己想 10 分钟。不要往下翻。**

### 场景：一个微型的宇宙

1970 年，英国数学家约翰·康威设计了一个"零玩家游戏"——只有规则，没有玩家。

一个无限大的网格，每个格子（细胞）非黑即白——要么活着（■），要么死了（□）。

每一代（帧），每个细胞根据**周围 8 个邻居**的状态，按照 4 条规则更新：

```
1. 活细胞，邻居 < 2 个活 → 孤独，死亡
2. 活细胞，邻居 2-3 个活 → 存活
3. 活细胞，邻居 > 3 个活 → 拥挤，死亡
4. 死细胞，邻居 = 3 个活 → 繁殖，变活
```

### 你要做的事

打开 Python，直接模拟一个小世界：

```text
初始状态（第一代）：
□ ■ □ □
□ □ ■ ■
□ ■ ■ □
□ □ □ □
```

**手动推演 3 代**：下一代的每个格子怎么变？

然后思考更本质的问题：

> **4 条规则，凭什么能产生"生命"？**
> ——在这里，"生命"不是生物意义上的，而是一些复杂的、自我组织的**模式**：
> 会移动的、会繁殖的、会吞噬其他模式的……甚至，能进行计算的。

> 💬 *先手动推演几代，感受规则的运作。然后想想：这些规则里没有"移动"这条指令——但会不会有一些结构，看起来像在"移动"？*

## 🪜 阶段二：提示链

> **给自己最后一次机会——先试再想 5 分钟。还是没思路？逐条看提示，看到哪条有感觉了就停下来继续想。**

**提示 1：按规则走一遍**

初始状态：

```text
  0 1 2 3
0 □ ■ □ □
1 □ □ ■ ■
2 □ ■ ■ □
3 □ □ □ □
```

看 (1,1)——这个是中间的活细胞。数它周围 8 格的活细胞数：

```text
(0,0)=□ (0,1)=■ (0,2)=□
(1,0)=□ (1,1)=■ (1,2)=■
(2,0)=□ (2,1)=■ (2,2)=■
```

8 个邻居中活的有 4 个（(0,1), (1,2), (2,1), (2,2)）。

规则 3：活细胞，>3 个活邻居 → 拥挤死亡。

于是 (1,1) 下一代变 □。

检查每个格子，你的下一代应该长这样：

```text
□ □ ■ □
□ □ □ ■
□ □ □ □
□ □ □ □
```

**提示 2：什么是"稳定"？**

有些图案推演几代后不再变化，这叫**静物**（Still Life）：

```text
□ □ □ □     □ □ □ □
□ ■ ■ □     □ ■ ■ □    ← 2×2 方块，永远不变
□ ■ ■ □     □ ■ ■ □
□ □ □ □     □ □ □ □
```

验证：每个活细胞有 3 个邻居（在 2-3 范围内），每个有 3 个邻居的死细胞... 这里没有死细胞有 3 个活邻居。所以不变。

有些图案会周期性重复——2 拍、3 拍、15 拍、甚至上百拍后恢复原状。这叫**振荡器**（Oscillator）。

**提示 3：有没有"会动"的东西？**

规则里根本没有"移动"这个词。但看这个 3×3 图案：

```text
第 1 代     第 2 代     第 3 代
□ ■ □      □ □ □      □ □ □
□ □ ■      ■ □ ■      □ ■ □
■ ■ ■      □ ■ ■      ■ ■ ■
□ □ □      □ □ ■      □ □ □
```

这是**滑翔机（Glider）**——每 4 代向右下角移动一格。它是一个"结构体"，不是单个细胞——整体向一个方向移动，但每个细胞都只按 4 条规则做局部决策。

```
没有"飞行"的指令，但出现了"飞行的模式"。
```

**提示 4：滑翔机能做什么？**

如果你把滑翔机看作"信息载体"——它在空间中移动，可以被创造、碰撞、湮灭、被"吃掉"。

这就是**计算的基础**：

```
滑翔机碰撞 → 产生新的滑翔机 → 类似信号复制
滑翔机被吃 → 信号被吸收
滑翔机与滑翔机枪碰撞 → 类似于逻辑门的输出
```

有人在生命游戏的世界里**搭建了 AND/OR/NOT 门、寄存器、时钟**，然后拼出了一台完整的计算机。这台计算机运行在生命游戏的网格上，每秒模拟几代，**能运行任意程序。**

更让人头皮发麻的是：有人在这台「生命游戏里的计算机」上，又跑了一个**生命游戏模拟器**。

## 🧠 阶段三：核心洞察 + 三步翻译法

### 核心洞察

> **四条规则，零条关于"移动"或"繁殖"的指令——但这些动态的、自组织的模式却自然涌现了。**
>
> 这就是"涌现"（Emergence）——微观上的简单规则，在宏观上产生不可预测的复杂性。
>
> 更深刻的是：**生命游戏是图灵完全的。** 你能在任何图灵机上解决的问题，都能在生命游戏的网格上解决（假设无限时间和空间）。

```
──────────────────────────────────────────────
微规则（每格每代看的）
  1. 活 < 2  → 死（孤独）
  2. 活 2-3  → 活（适存）
  3. 活 > 3  → 死（拥挤）
  4. 死 = 3  → 活（繁殖）

宏涌现（我们看到的）
  静物  → 不变的结构
  振荡器 → 周期性的变化
  滑翔机 → 整体"移动"的模式
  滑翔机枪 → 不断产生滑翔机的结构
  逻辑门 → AND/OR/NOT 信号处理
  计算机 → 图灵完全的通用计算
  递归模拟 → 在生命游戏里跑生命游戏
──────────────────────────────────────────────
```

### 三步翻译——自己写代码

**第一步：说清意图**

```
实现康威生命游戏的模拟器。
输入：网格大小（宽、高）、初始状态。
对每一代：
  计算每个细胞的 8 个邻居中活的数量。
  按 4 条规则更新所有细胞。
  输出当前代（终端打印即可）。
```

**第二步：拆成标准操作**

```
→ 定义生存规则函数：
    输入：当前细胞状态（活/死）、活邻居数 n
    输出：下一代的细胞状态
→ 对每个细胞：
    → 扫描 8 个方向（上、下、左、右、对角线）
    → 统计活邻居数量
    → 应用规则得到下一代
→ 一次性更新全部细胞（同步更新）
→ 打印网格
```

**第三步：匹配到代码**

| 标准操作 | 代码实现 |
|---------|---------|
| 扫描 8 个邻居 | `for di in [-1,0,1]: for dj in [-1,0,1]: skip (0,0)` |
| 边界处理 | `ni = (i + di) % n`（环形边界）或条件判断 |
| 规则逻辑 | `if grid[i][j] and n in (2,3): live; elif not grid[i][j] and n == 3: live; else: die` |
| 同步更新 | `new_grid = apply_rules(grid)` → `grid = new_grid` |
| 终端打印 | `print('■' if cell else '□', end='')` |

## ✅ 阶段四：参考实现 + 迁移 + 本周挑战

### 参考实现

```python
import os
import time


class GameOfLife:
    def __init__(self, width: int, height: int):
        self.width = width
        self.height = height
        self.grid = [[False] * width for _ in range(height)]

    def set_cell(self, x: int, y: int, alive: bool = True):
        """设置某个细胞的状态。"""
        self.grid[y][x] = alive

    def count_neighbors(self, x: int, y: int) -> int:
        """统计 (x, y) 周围 8 格的活细胞数量。"""
        count = 0
        for dy in (-1, 0, 1):
            for dx in (-1, 0, 1):
                if dx == 0 and dy == 0:
                    continue
                nx, ny = (x + dx) % self.width, (y + dy) % self.height
                if self.grid[ny][nx]:
                    count += 1
        return count

    def next_generation(self):
        """同步更新到下一代。"""
        new_grid = [[False] * self.width for _ in range(self.height)]

        for y in range(self.height):
            for x in range(self.width):
                n = self.count_neighbors(x, y)
                alive = self.grid[y][x]

                if alive and n in (2, 3):
                    new_grid[y][x] = True        # 规则 2：存活
                elif not alive and n == 3:
                    new_grid[y][x] = True        # 规则 4：繁殖
                # 其他情况均为死（规则 1 和 3）

        self.grid = new_grid

    def display(self):
        """终端打印当前代。"""
        os.system('cls' if os.name == 'nt' else 'clear')
        for row in self.grid:
            print(''.join('■' if cell else '□' for cell in row))

    def run(self, generations: int, delay: float = 0.2):
        """运行指定代数，每代之间暂停 delay 秒。"""
        for _ in range(generations):
            self.display()
            self.next_generation()
            time.sleep(delay)


# ── 测试：滑翔机（Glider）──
def glider_demo():
    game = GameOfLife(20, 20)
    # 滑翔机初始图案
    glider = [(1, 0), (2, 1), (0, 2), (1, 2), (2, 2)]
    for x, y in glider:
        game.set_cell(x, y, True)
    game.run(40)


# ── 测试：滑翔机枪（Gosper Glider Gun）──
# 一个不断产生滑翔机的结构——相当于一个"信号发报机"
def glider_gun_demo():
    game = GameOfLife(40, 30)

    # Gosper 滑翔机枪坐标（经过验证的起始点）
    gun_cells = [
        (24, 0), (22, 1), (24, 1),
        (12, 2), (13, 2), (20, 2), (21, 2), (34, 2), (35, 2),
        (11, 3), (15, 3), (20, 3), (21, 3), (34, 3), (35, 3),
        (0, 4), (1, 4), (10, 4), (16, 4), (20, 4), (21, 4),
        (0, 5), (1, 5), (10, 5), (14, 5), (16, 5), (17, 5), (22, 5), (24, 5),
        (10, 6), (16, 6), (24, 6),
        (11, 7), (15, 7),
        (12, 8), (13, 8),
    ]

    for x, y in gun_cells:
        game.set_cell(x, y, True)

    game.run(200, delay=0.05)


if __name__ == "__main__":
    # 先看滑翔机
    glider_demo()
```

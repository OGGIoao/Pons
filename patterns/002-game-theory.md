## 🚩 阶段一：挑战

> **只看这个阶段，自己想 10 分钟。不要往下翻。**

### 游戏规则

你和朋友在咖啡厅，餐巾纸上画了一个 **3×3 的棋盘**。一枚硬币放在左上角格子。规则很简单：

- 两人轮流移动硬币
- 每次可以 **向右走 1 格**、**向下走 1 格**、或者 **沿对角线走 1 格**（同时向右和向下）
- **先走到右下角** 的人获胜

朋友说："你先手。"

### 问题

> 你先手，能保证赢吗？——不能的话，为什么？能的话，第一步走哪？

> 然后：**能不能用一个系统性的方法，判断任意大小棋盘上先手的胜负？**

## 🪜 阶段二：提示链

> **给自己最后一次机会——先试再想 5 分钟。还是没思路？逐条看提示，看到哪条有感觉了就停下来继续想。**

**提示 1：站在谁的视角？**

谁站在右下角的格子 (3,3) 上？——没人。因为走到 (3,3) 的人已经赢了。所以如果轮到你走的时候你在 (3,3)，说明对手已经赢了——**你输了**。

所以 **(3,3) 是必败**。从这个已知结论开始往前推。

**提示 2：逆推第一步**

考察 (3,2)：你在 (3,2)，轮到你走。你走 → (3,3)——对方站在 (3,3)，而轮到对方时站在 (3,3) 意味着对方输了。所以 **你赢了**。

结论：**站在 (3,2) 的人必胜**——因为他能一步走到必败态。

同样地，(2,3) 也能一步到 (3,3)。(2,2) 能斜走到 (3,3)。

**提示 3：核心定义的递推**

```
必胜态：存在至少一条走法，能到达一个必败态
必败态：所有可能的走法，都只能到达必胜态
```

从右下角开始，一格一格往左上角标记——每个格子要么是 **N**（必胜，Next player wins），要么是 **P**（必败，Previous player wins）。

```
(3,3) → P（已知）
(3,2) → 能走到 (3,3)[P]  →  N
(2,3) → 能走到 (3,3)[P]  →  N
(2,2) → 能走到 (2,3)[N], (3,2)[N], (3,3)[P]  →  存在到 P 的路  →  N
(3,1) → 只能走到 (3,2)[N]  →  所有走法都到 N  →  P
(1,3) → 只能走到 (2,3)[N]  →  所有走法都到 N  →  P
……继续往上推
```

**提示 4：你已经见过这个模式了**

001 卡特兰数的递推：已知小的（C₀），算大的（Cₙ）。
002 博弈的逆推：已知终局（P），算起点的（胜或负）。

方向相反，但思想完全一样——**用已知状态推导未知状态**。只不过这次是在棋盘网格上递推，而不是在一维数组上。

## 🧠 阶段三：核心洞察 + 三步翻译法

### 核心洞察

> **从终局向后倒推——每个状态要么是"必胜"（能走到一个必败态），要么是"必败"（所有走法都通向必胜态）。**
>
> 这是所有回合制博弈问题的"牛顿第二定律"——统一的底层框架。不管是五子棋、象棋、围棋，还是商业谈判——只要你能定义状态和走法，这个框架就适用。

把整张棋盘看作一个递推系统：

```
P(n,m) = 必败          （终点没人能赢，站在终点的人输了）
P(i,j) = 必胜  如果存在一步走到某个必败位置
P(i,j) = 必败  如果所有走法都只能走到必胜位置
```

### 三步翻译——自己写代码

**第一步：说清意图**

```
输入棋盘大小 n×m，输出每个格子是必胜还是必败。
用逆推法从右下角往左上角递推计算。
```

**第二步：拆成标准操作**

```
→ 创建一个 (n+2)×(m+2) 的二维数组，全部初始化为 True（必胜）
→ 右下角 (n,m) 设为 False（必败）
→ 从右下角向左上角遍历每个格子，跳过 (n,m)：
    → 检查右、下、斜三个方向
    → 如果存在至少一个是 False（必败）→ 当前格子为 True（必胜）
    → 否则为 False（必败）
→ 返回数组
```

**第三步：匹配到代码**

| 标准操作 | 代码实现 |
|---------|---------|
| 创建二维数组 | `dp = [[True] * (m + 2) for _ in range(n + 2)]` |
| 终局标记 | `dp[n][m] = False` |
| 逆序遍历 | `for i in range(n, 0, -1): for j in range(m, 0, -1):` |
| 决策判断 | `dp[i][j] = not (dp[i+1][j] and dp[i][j+1] and dp[i+1][j+1])` |

## ✅ 阶段四：参考实现 + 迁移 + 本周挑战

### 参考实现

**写法 A——通用逆推求解器（理解思想）**

```python
def solve_board(n: int, m: int) -> list[list[bool]]:
    """
    返回 n×m 棋盘每个格子的胜负状态。
    True  = 必胜（N-position，轮到的人能赢）
    False = 必败（P-position，轮到的人会输）
    """
    dp = [[True] * (m + 2) for _ in range(n + 2)]
    # 越界的一圈自动为 True（不影响结果）
    dp[n][m] = False  # 终点：必败

    for i in range(n, 0, -1):
        for j in range(m, 0, -1):
            if i == n and j == m:
                continue
            # 三个方向：右、下、斜
            # 只要有一个是必败 → 当前必胜
            dp[i][j] = not (dp[i+1][j] and dp[i][j+1] and dp[i+1][j+1])
    return dp


def print_board(dp: list[list[bool]], n: int, m: int):
    """打印棋盘，N=必胜，P=必败"""
    print("   | " + " ".join(f"c{j}" for j in range(1, m + 1)))
    print("---+" + "---" * m)
    for i in range(1, n + 1):
        row = f"r{i} | " + " ".join(
            "P" if not dp[i][j] else "N" for j in range(1, m + 1)
        )
        print(row)


# 测试 3×3
dp = solve_board(3, 3)
print_board(dp, 3, 3)

# --- 输出 ---
#    | c1 c2 c3
# ---+---------
# r1 | P  N  P
# r2 | N  N  N
# r3 | P  N  P
#
# (1,1) 是 P → 先手必败！
```

**写法 B——公式（一秒看出答案）**

如果你推完了 3×3 棋盘，你可能发现了规律。推广到任意 n×m：

```python
def can_first_win(n: int, m: int) -> bool:
    """先手能否在 n×m 棋盘上必胜？"""
    return not ((n - 1) % 2 == 0 and (m - 1) % 2 == 0)


def is_p_position(n: int, m: int, i: int, j: int) -> bool:
    """位置 (i,j) 是必败态吗？"""
    return (n - i) % 2 == 0 and (m - j) % 2 == 0
```

**规律**：必败态 (P) 出现在 **到右边缘距离 (n-i) 和到下边缘距离 (m-j) 都是偶数** 的位置。

**为什么？** 从必败态出发，无论走右、下、还是对角线，都会让至少一个距离变成奇数，从而进入必胜态。从必胜态，总有办法（走对角线或单方向）把两个距离都变回偶数，把对手送回必败态。

所以先手在 **奇数 × 奇数** 的棋盘上必败，在 **至少一边为偶数** 的棋盘上必胜。

### 这个模式还出现在……

| 领域 | 描述 | 对应关系 |
|------|------|---------|
| **取石子游戏 (Nim)** | 多堆石子，每轮取若干颗 | 把石子数看作"状态"，逆推逻辑完全相同 |
| **棋类 AI** | 极小化极大搜索 (Minimax) | +∞ 代替"必胜"，-∞ 代替"必败"，就是 AlphaGo 的搜索树 |
| **动态规划** | 任何具有最优子结构的问题 | 必胜/必败是 DP 的一个特例——状态转移是"存在性"而非"最值" |
| **日常博弈** | 商业谈判、拍卖竞价 | 从终局（成交/流产）倒推对手每一步的"必胜/必败"选择 |

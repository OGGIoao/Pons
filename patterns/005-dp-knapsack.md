## 🚩 阶段一：挑战

> **只看这个阶段，自己想 10 分钟。不要往下翻。**

### 场景：背包客的精算

你要进山徒步 5 天。背包容积上限是 **7 公斤**。桌上 5 件装备：

| 物品 | 重量 (kg) | 价值 (分) |
|------|-----------|-----------|
| 帐篷 ⛺ | 3 | 10 |
| 睡袋 🛌 | 4 | 12 |
| 炉头 🔥 | 2 | 4 |
| 净水器 💧 | 3 | 8 |
| 压缩干粮 🍞 | 1 | 3 |

**规则：每件物品要么拿走，要么留下——不能只装一半。**

### 问题

**在不超过 7 公斤的前提下，怎么装才能让总"价值"最高？**

> 💬 *别急着算。先想一个问题："如果我用"每公斤价值最高"的贪心法——先装每公斤最值钱的，能不能得到最优解？试一下。*

> *找到了最优方案后，再问自己：如果物品有 100 件，暴力枚举 2¹⁰⁰ 种组合不可能——有没有一个系统性的方法？*

## 🪜 阶段二：提示链

> **给自己最后一次机会——先试再想 5 分钟。还是没思路？逐条看提示，看到哪条有感觉了就停下来继续想。**

**提示 1：先试试"每公斤价值最高"**

计算每件物品的"价值/重量"比：

```
睡袋 3.0/公斤  帐篷 3.33/公斤  净水器 2.67/公斤
炉头 2.0/公斤  干粮 3.0/公斤
```

按这个顺序装：帐篷(3) + 睡袋(4) = **7 公斤，22 分**。

但你发现另一组合：帐篷(3) + 净水器(3) + 干粮(1) = **7 公斤，21 分**——更差。

有没有更好的方案？**净水器(3) + 睡袋(4) = 7 公斤，20 分**——也不如帐篷+睡袋。

所以贪心好像对了？再试一件物品：假设多一个**急救包(1kg, 价值 2)**。贪心改选帐篷(3)+睡袋(4)+... 7kg 满了。但帐篷(3)+净水器(3)+急救包(1)=7kg，20 分，不如原来的帐篷+睡袋。贪心似乎一直对……等等，让我换个场景。

假设有这些物品：

| 物品 | 重量 | 价值 |
|------|------|------|
| A | 4 | 10 |
| B | 3 | 8 |
| C | 2 | 6 |
| D | 2 | 4 |

容量 = 5。价值/重量比：A=2.5, B=2.67, C=3, D=2。

贪心选 C(2,6) + B(3,8) = 5kg，14 分。
但 A(4,10) + D(2,4) = 6kg 超重。A(4,10) + C(2,6) = 6kg 超重。
A(4,10) + D... 不能同时选（6kg > 5）。

最优是 B(3,8) + C(2,6) = 14 分 —— 和贪心一样。

这还不是一个让贪心失败的例子。让我设计一个：

| 物品 | 重量 | 价值 |
|------|------|------|
| A | 5 | 14 |
| B | 4 | 11 |
| C | 3 | 9 |
| D | 3 | 8 |

容量 = 6。价值/重量：A=2.8, B=2.75, C=3.0, D=2.67。
贪心选 C(3,9)，剩下 3kg，可以选 D(3,8) = 6kg, 17 分。
但 B(4,11) + D(3,8) = 7kg 超重。A(5,14) + ...剩下 1kg 没有 = 14 分。B(4,11) + C(3,9) = 7kg 超重。

最优是 C(3,9) + D(3,8) = 17 分。贪心依然对了……

其实经典的贪心反例需要"大而重但价值平平"的物品：

| 物品 | 重量 | 价值 |
|------|------|------|
| A | 4 | 10 |
| B | 3 | 7 |
| C | 2 | 4 |
| D | 2 | 4 |

容量 = 4。价值/重量：A=2.5, B=2.33, C=2, D=2。
贪心选 A(4,10) = 10 分，满了。
但 B(3,7) + C(2,4) = 5kg 超重。B(3,7) + D(2,4) = 5kg 超重。
C(2,4) + D(2,4) = 4kg, 8 分 < 10。

还是 A 优。这例子的问题是贪心也选了最优……这很正常，因为贪心反例需要特定结构。

让我用一个真正能让贪心失败的标准反例：

容量 = 5

| 物品 | 重量 | 价值 |
|------|------|------|
| A | 4 | 10 |  (2.5/kg)
| B | 3 | 7 |   (2.33/kg)
| C | 2 | 5 |   (2.5/kg)

贪心按价值/重量：A(4,10)，剩 1kg 不能装 → 10 分。
C(2,5) + B(3,7) = 5kg, 12 分 > 10 分。

**这个反例成立！** 贪心选了最高的价值/重量比 A，但剩下 1kg 被浪费了。而 C+B 虽然单件价值/重量不比 A 高，但组合起来利用了全部容量。

```
贪心（按价值/重量比）→ A(4,10) = 10 分 ❌
最优组合           → B(3,7) + C(2,5) = 12 分 ✅
```

**提示 1：贪心失效**

如果按"每公斤价值最高"的顺序装：A(4kg,10分) 装进去，剩下 1kg 什么也装不了 → 10 分。

但 B(3kg,7分) + C(2kg,5分) = **12 分**。

贪心法在"整数背包"（不能切割）中会失效——因为装下大件后留下的小空隙可能浪费。

**提示 2：决策的岔路口**

站在每件物品面前只有两个选择：**拿**或**不拿**。

拿 → 占一些重量，加一些价值。不拿 → 重量和价值不变。

但关键是：拿或不拿的决策，**取决于剩下的容量还能装什么**。

```
面对物品 A（4kg, 10分）：
  拿 → 用掉 4kg，剩下 1kg 容量
  不拿 → 0kg 占用，留下 5kg 容量给 B,C 选
```

直觉：如果知道"剩下 1kg 最多能装多少分"，和"剩下 5kg 最多能装多少分"，就能决定拿不拿 A。

**提示 3：子问题定义**

定义 `dp[i][c]` = **从前 i 件物品中选，总容量不超过 c，能得到的最大价值**。

物品列表：1=A(4,10), 2=B(3,7), 3=C(2,5)

```
dp[0][c] = 0    （没有物品可选，价值为 0）
dp[i][0] = 0    （没有容量，什么也装不了）
```

面对第 i 件物品、当前容量 c：
- **不拿** → dp[i][c] = dp[i-1][c]（跳过它，继承前 i-1 件的最优）
- **拿** → 前提：w_i ≤ c。拿完后剩余容量 c-w_i，前 i-1 件的最优是 dp[i-1][c-w_i]

所以：

```
dp[i][c] = max(dp[i-1][c], dp[i-1][c-w_i] + v_i)   如果 w_i ≤ c
dp[i][c] = dp[i-1][c]                                如果 w_i > c
```

**提示 4：填表**

容量 c 从 0 到 5，物品从 1 到 3：

```
容量:   0  1  2  3  4  5
dp[0]: [0, 0, 0, 0, 0, 0]     ← 无物品
dp[1]: [0, 0, 0, 0,10,10]     ← 考虑 A(4,10) 后
dp[2]: [0, 0, 0, 7,10,10]     ← 考虑 B(3,7) 后
dp[3]: [0, 0, 5, 7,10,12]     ← 考虑 C(2,5) 后  ✅ dp[3][5]=12
```

最后一行最后一列就是答案：dp[3][5] = 12。

## 🧠 阶段三：核心洞察 + 三步翻译法

### 核心洞察

> **一个最优解，要么包含第 i 件物品，要么不包含。如果包含——剩余容量的子问题也必须是最优的。这叫"最优子结构"。**

```
面对第 i 件物品：
  拿？→ 问题变小：容量减 w_i，物品剩 i-1 件
  不拿？→ 问题变小：容量不变，物品剩 i-1 件
  
dp[i][c] = max(dp[i-1][c], dp[i-1][c-w_i] + v_i)
```

这一条公式是 001（递推）的推广、002（倒推）在不同维度上的应用。

**什么不是背包 DP？**——如果每件物品可以切一半（比如面粉），那就是"分数背包"，贪心就好了。不能切才是难点。

### 三步翻译——自己写代码

**第一步：说清意图**

```
输入物品列表（重量、价值）和背包容量 C，
输出能装的最大总价值。
```

**第二步：拆成标准操作**

```
→ 创建二维数组 dp[n+1][C+1]，全部初始化为 0
→ 对于每个物品 i（从 1 到 n）：
    → 对于每个容量 c（从 0 到 C）：
        → 如果 w_i > c：不能拿 → dp[i][c] = dp[i-1][c]
        → 否则：拿或不拿取最大值
→ 返回 dp[n][C]
```

**第三步：匹配到代码**

| 标准操作 | 代码实现 |
|---------|---------|
| 创建二维表 | `dp = [[0] * (C + 1) for _ in range(n + 1)]` |
| 遍历物品 | `for i in range(1, n + 1):` |
| 遍历容量 | `for c in range(C + 1):` |
| 不拿 | `dp[i][c] = dp[i-1][c]` |
| 拿（条件） | `if w_i <= c: dp[i][c] = max(dp[i][c], dp[i-1][c-w_i] + v_i)` |

## ✅ 阶段四：参考实现 + 迁移 + 本周挑战

### 参考实现

**写法 A——二维 DP（最直观）**

```python
def knapsack_2d(weights: list[int], values: list[int], C: int) -> int:
    """
    0/1 背包，返回最大总价值。
    weights, values: 长度 n 的物品重量和价值列表
    C: 背包容量
    """
    n = len(weights)
    dp = [[0] * (C + 1) for _ in range(n + 1)]

    for i in range(1, n + 1):
        w_i = weights[i - 1]
        v_i = values[i - 1]
        for c in range(C + 1):
            if w_i > c:
                dp[i][c] = dp[i - 1][c]          # 不能拿
            else:
                # 拿或不拿，取最大值
                dp[i][c] = max(dp[i - 1][c],
                               dp[i - 1][c - w_i] + v_i)
    return dp[n][C]


# 测试：贪心反例
weights = [4, 3, 2]
values  = [10, 7, 5]
print(knapsack_2d(weights, values, 5))  # → 12
```

**写法 B——一维 DP（省空间，面试常考）**

观察递推式：dp[i][c] 只依赖 dp[i-1][...]——上一行的数据。所以只需要一维数组，从右往左更新。

```python
def knapsack_1d(weights: list[int], values: list[int], C: int) -> int:
    """一维数组优化版"""
    dp = [0] * (C + 1)

    for w_i, v_i in zip(weights, values):
        # 从右往左更新：否则 dp[c - w_i] 是本轮已更新的值（不是上一轮的）
        for c in range(C, w_i - 1, -1):
            dp[c] = max(dp[c], dp[c - w_i] + v_i)

    return dp[C]


print(knapsack_1d(weights, values, 5))  # → 12
```

**为什么从右往左？** 如果从左往右，dp[c-w_i] 可能已经被本轮物品更新过了，再用它等于"同一件物品拿了多次"——那就变成了"完全背包"。从右往左确保 dp[c-w_i] 还是上一轮的旧值。

### 这个模式还出现在……

| 领域 | 问题 | 对应关系 |
|------|------|---------|
| **资源分配** | 预算有限，选项目组合使总收益最大 | 预算 = 容量，项目利润 = 价值 |
| **芯片设计** | 芯片面积固定，放不同功能模块 | 面积 = 重量，模块效能 = 价值 |
| **货运优化** | 卡车载重有限，装载不同货物 | 载重 = 容量，运费 = 价值 |
| **投资组合** | 资金固定，分散投资 | 投入资金 = 重量，预期回报 = 价值 |

**变体对比：**

| 变体 | 规则 | 递推差异 |
|------|------|---------|
| **0/1 背包**（本文） | 每件最多拿一次 | 一维从右往左更新 |
| **完全背包** | 每件可以拿无限次 | 一维从左往右更新 |
| **多重背包** | 每件有有限件数 | 二进制拆分 → 0/1 背包 |
| **分数背包** | 可以切一部分 | 贪心，按价值/重量排序即可 |

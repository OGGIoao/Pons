# 便携翻译器 · Portable Refcard

> **在任何电脑上，只要有 Python 就能工作。**
> 核心技能只有一项——**三步翻译法**，把"我要做什么"翻译成"用什么函数"。
>
> 本文件提供通用翻译框架 + 各领域的意图→API 映射示例。
> 当你面对一个新领域时，用同样的方法自己构建映射。

---

## 第一部分：三步翻译法（通用技能）

> 这是脑桥的**元技能**。掌握它，你就不需要背任何 API。

### 步骤 1：把中文意图说清楚

```
❌ "我想画一张好看的图"
✅ "我想在 [-3, 3] 区间内画一条正弦曲线，x 轴标为 '时间'，y 轴标为 '振幅'"
```

```
❌ "我想分类这些数据"
✅ "我想训练一个模型，输入是 10 维特征，输出是 3 个类别之一"
```

> **判断标准**：如果这句话能说给另一个程序员听并且他直接能写代码——说清楚了。
> 否则回去重新说。

### 步骤 2：拆解成标准操作

```
❌ "画一条曲线"
✅ "生成数据 → 创建画布 → 调用 plot 函数 → 设置标签 → 显示"
```

```
❌ "做个分类"
✅ "拆分数据集 → 初始化模型 → 喂数据训练 → 预测 → 评估准确率"
```

> **判断标准**：每一步都以一个动词开头——"生成""创建""调用""设置"。
> 每步对应一个函数调用或一个操作。

### 步骤 3：匹配到实际函数

```
"生成数据"   → np.linspace() / np.arange()
"创建画布"   → plt.subplots()
"画曲线"     → ax.plot()
"设标签"     → ax.set_xlabel()
"显示"       → plt.show()
```

> **判断标准**：如果某一步找不到对应的函数——说明拆得还不够细，或者这一步用标准库做不了（需要自己写逻辑）。

## 第二部分：各领域映射示例

> 以下映射都是**示例**——展示三步翻译法在不同领域的样子。

### 数学可视化

| 意图 | 技术翻译 | 代码 |

| "取一段连续区间" | "等间距采样" | `np.linspace(a, b, n)` |

| "算复数模长" | "算绝对值" | `np.abs(Z)` |
| "求部分和" | "逐项累加" | `np.cumsum(terms)` |

### 数据结构与算法

## 第三部分：通用的"七行 Python"

这 4 个 import 覆盖了脑桥 90% 的卡片：

```python
import numpy as np
import matplotlib.pyplot as plt
from collections import deque
import bisect
```

再补两个就覆盖 95%：

```python
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score
```

## 第四部分：三步翻译法 × 四阶段卡片 = 完整学习流

```
三步翻译法是 "我知道了，怎么写？"
四阶段卡片是  "我不知道往哪想……哦，原来是这个思路！"

组合起来：
  四阶段卡片给你「思路」
  三步翻译法给你「把思路变成代码的能力」
```

所以学卡片的时候：
- **阶段三**指出核心思路后，**不要看参考实现**
- 自己用三步翻译法把思路翻译成代码
- 写完了再去阶段四对照参考实现
